Configurar subida + 1º subida
     (Dentro de la carpeta que le hagamos git init)

  1. git init -> Iniciar repositorio o clonar de alguien
    2. 

  2. git add nombre.extensión // git add --all -> Añadimos de Working Dyrectory -> a Staging Area
    git status (Añadido (git add) / No añadido (no git add) -> Si modificamos archivos se pondrá en verde la parte anterior y en rojo lo nuevo el mismo archivo, hasta que hagamos git add)
    3. 

  3. git commit -m "Comentario" // git commit (Staging Area -> nos abre el editor de texto -> Podemos escribir dentro, para salir y guardar (ESC + :wq!) (Pasar a Local Repository))
    4. 
    git log -> Log (Autor / Día y hora / Comentario escrito -> Arriba (Los más nuevo) - Abajo)(Lo más viejos)
    
  4. git remote add nombreDelRepositorio https://github.com/usuario/nombre-del-proyecto -> (nombreDelRepositorio o origin o upstream). Local Repository: Ejemplo
    git remote -v -> Comprobar si está bien subido
    5. 

  5. git push -u nombreDelRepositorio master -> Subir el proyecto relacionado con el repositorio. Nos pide identificarnos
Obtener proyecto
git pull nombreDelRepositorio master -> Obtener archivos de Github de otros usuarios que trabajan en el mismo proyecto o modificados por uno mismo desde la web
